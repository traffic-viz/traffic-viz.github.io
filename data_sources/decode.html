How to decode ADS-B and Mode S data?

class traffic.data.adsb.decode.ModeS_Decoder(reference=None)

This class implements the full data decoding process with the pyModeS library

Data is first available as a list of messages per aircraft, but can be exported as a Traffic object.

Warning

The from_address(), from_dump1090(), and from_rtlsdr() classmethods start a decoding thread on the creation of the object. The thread can be stopped with a decoder.thread.stop() call.

Parameters

reference (Union[None, str, Airport, tuple[float, float]]) –

A reference location must be provided to decode ground messages. A reference can be set as:

  • a tuple of latitude/longitude coordinates

  • a str calling for an Airport structure

If no reference is provided (None), then reference location (latitude=0, longitude=0) is considered until more data can be used to readjust the reference.

classmethod from_address(host, port, reference, file_pattern='~/ADSB_EHS_RAW_%Y%m%d_tcp.csv', time_fmt='radarcape', uncertainty=False)

Decode raw messages transmitted over a TCP network.

The file should contain for each line at least a timestamp and an hexadecimal message, as a CSV-like format.

Parameters

  • host (str) – the IP address of the host to connect

  • port (int) – the port of the host to connect

  • reference (Union[str, Airport, tuple[float, float]]) – the reference location, as specified above

  • file_pattern (str) –

    the filename where to dump received hexadecimal messages

    Timestamp format specifiers are accepted.

    Default value: "~/ADSB_EHS_RAW_%Y%m%d_tcp.csv"
    (The ~ character gets expanded as your home directory)

  • time_fmt (str) –

    (default: "radarcape")

    • if set to "radarcape", timestamp information included in the long message is assumed to be GPS adjusted and is therefore used to adjust the timestamp of each message;

    • if set to "dump1090", clock information provided by the software is used to adjust the timestamp of each message, but the first timestamp is based on the computer clock;

    • if set to "default", the timestamp from the computer is used for all decoded messages

  • uncertainty (bool) – if True, decode also uncertainty information

Return type

ModeS_Decoder

classmethod from_file(filename, reference, uncertainty=False, template='time, longmsg', sep=',')

Decode raw messages dumped in a text file.

The file should contain for each line at least a timestamp and an hexadecimal message, as a CSV-like format.

Parameters

  • filename (str | Path) – the path to the file containing the data

  • reference (str | Airport | tuple[float, float]) – the reference location, as specified above

  • uncertainty (bool) –

    if True, decode also uncertainty information

  • template (str) –

    the header explaining how data is organised

    Three parameters are accepted:

    • time represents the timestamp in seconds (float)

    • shortmsg represents the regular version of the ADS-B hexadecimal message (messages of length 28 for ADS-B)

    • longmsg represents messages containing timestamp information as a prefix, as dumped by many decoding softwares, such as dump1090 or other receivers.

    By default, the expected format is time, longmsg

Return type

ModeS_Decoder
